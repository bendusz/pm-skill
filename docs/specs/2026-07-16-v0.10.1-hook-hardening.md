# v0.10.1 — hook hardening design

Date: 2026-07-16
Status: approved
Origin: whole-codebase codex review (gpt-5.6-sol), 6 findings — 5 major, 1 minor.

## Goal

Make the shell hooks' mechanical enforcement match what the documentation promises: correct
project-root discovery, traversal/symlink-safe path checks, a credible secret tripwire, globally
unique actor ids, a value-based outgoing-diff secret scan, and behavioral tests that pin all of it.

## Shared infrastructure: `hooks/lib.sh`

The four hooks currently duplicate root discovery, path classification, and actor-id derivation —
the fixes below would triple that duplication, so they move into one sourced library
(`plugins/pm-skill/hooks/lib.sh`, executable like the rest). Each hook sources it via its own
directory (`$(dirname "$0")/lib.sh`) and stays fail-open: if the lib is missing or unreadable,
`exit 0`.

### F1 — project-root discovery (`pm_root`)

Root = first hit of:
1. `$CLAUDE_PROJECT_DIR` — if set, absolute, and a directory (the documented hook variable);
2. `git -C "$cwd" rev-parse --show-toplevel` — when inside a work tree;
3. `$cwd` unchanged (PM projects without git keep working).

All hooks then look for `pm/pm-state.json` (and the legacy `tmp/` fallback) under that root, so
launching Claude Code from `repo/packages/foo` no longer disables enforcement.

### F2 — canonical containment (`pm_relpath`)

Replaces the lexical `case "$file" in "$cwd"/*)` checks:
1. Canonicalize the root (`cd … && pwd -P`).
2. For the (possibly not-yet-existing) target: split into deepest **existing** ancestor +
   remainder; canonicalize the ancestor the same way; reject a remainder containing `..` or a
   symlinked escape (the canonicalized ancestor must still sit under the canonical root).
3. Return the normalized root-relative path for classification, or fail (treated by callers as
   "outside the tree").

`pm/../src/app` therefore classifies as `src/app` (blocked pre-sign-off), and a symlink under an
allowed prefix cannot smuggle a write outside it. Written in portable bash (no `realpath -m`,
which macOS lacks).

### F3 — credential-assignment tripwire (`pm_secret_scan`)

Two pattern groups, moved into the lib so F5 can reuse them:
- **Token formats** (unchanged, case-SENSITIVE — `AKIA…`, PEM headers, `gh[pousr]_…`,
  `github_pat_…`, `xox[baprs]-…`, `sk-…`, `AIza…`, JWT pairs).
- **Credential assignments** (now case-INSENSITIVE, quoted or unquoted values):
  key `(api[_-]?key|secret|token|passw(or)?d|credential)`, separator `:` or `=`, value = 8+ chars
  of `[A-Za-z0-9_/+=.\-]` — with placeholder exclusions preserved: values beginning `$`, `<`, or
  `{` (env refs, `<placeholder>`, templates) never trip, keeping the guard a high-confidence
  tripwire rather than a scanner. `API_KEY=abcdefghijklmno`, `Token: hunter2hunter2`, and
  uppercase quoted forms all now block.

`pm_secret_scan` reads stdin, exits 1 on a hit (prints the first matching line to stderr with the
value masked).

### F4 — globally unique actor ids

Derivation becomes: slug of the **full** git `user.email` (`v.bende@gmail.com` →
`v-bende-gmail-com`); fallback slug of `user.name`; last resort `unknown-actor`. One
implementation in the lib (`pm_actor_id`), used by both hooks; `logging-and-state.md` documents
the same rule for commands. Readable, deterministic, and cross-domain collisions
(`alex@a.com` vs `alex@b.com`) become distinct ids.

**Migration:** pre-0.10.1 actor files (local-part ids) show up as orphans; `/pm-skill:doctor`
already flags them, and `logging-and-state.md` + CHANGELOG gain a note: on first resume after
upgrading, rename `pm/actors/<old>.json`/`.HANDOFF.md` to the new id (one `git mv` each) — the
session-context hook's "no actor file for you yet" line points at resume.

### F5 — outgoing-diff secret scan

`implementation-loop.md` step 5 stops grepping for uppercase *labels* and instead pipes the exact
outgoing diff through the same value patterns:
`git diff <range> | ${CLAUDE_PLUGIN_ROOT}/hooks/lib.sh scan` (the lib doubles as a CLI:
`lib.sh scan` runs `pm_secret_scan` on stdin). Prefer a real scanner when one is installed
(`gitleaks`/`trufflehog`) — the reference says so — and on any hit the diff is NOT sent out.
`APIClient` in code no longer false-positives; lowercase real values no longer false-negative.

### F6 — behavioral hook tests

New `scripts/test-hooks.sh` (wired into `scripts/validate.sh` as a final check and run by CI):
table-driven fixtures that build throwaway projects under `mktemp -d`, pipe hook-shaped JSON to
each hook, and assert the exit code. Coverage at minimum:
- require-signoff: block pre-sign-off implementation write; allow docs/pm/tmp writes; allow when
  signed off; **subdirectory cwd with `CLAUDE_PROJECT_DIR`**; **subdirectory cwd with git
  fallback**; **`pm/../src` traversal blocks**; **symlinked dir escape blocks**; malformed JSON
  allows (fail-open); kill switch allows.
- pm-secrets-guard: blocks `API_KEY=…` unquoted, uppercase quoted, `Password: …`, and each token
  format; allows placeholders (`token = "<rotate me>"`, `KEY=$ENV_REF`) and prose; only guards
  `pm/`.
- actor-guard: same-actor write allows; other-actor blocks; **cross-domain ids distinct**;
  no-identity allows.
- session-context: emits pointer in PM projects from a subdirectory; silent in non-PM dirs.

## Out of scope

- No changes to hook *policy* (what is allowed/blocked when working as designed).
- No automatic migration of old actor files (documented manual rename via resume).
- codex-review/codex-help commands are untouched.
