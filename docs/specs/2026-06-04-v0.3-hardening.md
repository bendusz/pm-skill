## 1. Goal
Make the workflow's guarantees real (not just prose), recoverable, contributable, and learnable ‚Äî
without breaking the "generic, self-contained, fail-open" posture.

## 2. Scope (the user runs the real-life test #1; we build #2‚Äì#7)
- **#2 Worked example** ‚Äî a filled-in `examples/` walkthrough.
- **#3 Sign-off enforcement hook** ‚Äî a fail-open `PreToolUse` hook that blocks implementation writes

- **#5 Story-readiness check** ‚Äî a gate before a story is build-ready.

## 3. #3 ‚Äî The sign-off hook (designed for safety)
A plugin hook fires in **every** session of anyone who installs the plugin, so it MUST be harmless
outside a PM project. Design:
- `hooks/hooks.json` registers a `PreToolUse` hook on `Write|Edit|MultiEdit` ‚Üí
  `bash ${CLAUDE_PLUGIN_ROOT}/hooks/require-signoff.sh`.
- The script is **FAIL-OPEN**: it allows (exit 0) on any uncertainty ‚Äî kill switch
  `PM_SKILL_NO_ENFORCE=1`, no `jq`, no `tmp/pm-state.json`, unparseable JSON, or a target outside
  the project tree.
- It **blocks** (exit 2 + stderr reason) only when *all* hold: a `tmp/pm-state.json` exists with
  `signed_off=false`, AND the write targets a path **inside** the project that is **not** a planning
  artifact (allowlist: `docs/`, `tmp/`, `.git/`, `CLAUDE.md`, `.gitignore`).
- Net effect: for a normal user not running the PM skill, the hook is inert. Within a PM project it
  enforces "no implementation before sign-off" mechanically. Turns the v0.1 procedural gate into a
  real one (closes the roadmap item).

## 4. #4 ‚Äî State file + resume
`tmp/pm-state.json` (gitignored) is the machine-readable companion to `tmp/log.md`:
```json
{ "project": "...", "phase": "discovery|planning|decomposition|implementation|done",
  "signed_off": false, "approver": null, "approved_date": null,
  "integration_branch": "main", "current_sprint": null, "total_sprints": null,
  "current_story": null, "current_story_status": null, "branch": null,
  "next": "...", "updated": "YYYY-MM-DD HH:MM" }
```
- The skill creates it (signed_off=false) when planning begins and flips `signed_off=true` (+ approver
  + date) at the sign-off gate ‚Äî which is also what the hook reads.
- `/pm-skill:resume` reads it + `tmp/log.md` and continues. `SKILL.md` "On resume" points at both.
- Template: `templates/pm-state.json.template`.

## 5. #5 ‚Äî Story-readiness check
Before dispatching the builder, the story must pass a readiness check (decomposition.md +
implementation-loop step 0): testable acceptance criteria present, self-contained context present,
and a verification command present. If not ready, fix the story first ‚Äî don't build.

## 6. #7 ‚Äî Reviewer-finding triage
Before sending review findings to the builder (implementation-loop fix step), the PM **dedupes
across lenses and drops false positives / out-of-scope items**, then forwards only real
`block`/`major` findings. Keeps the bounded fix loop from burning rounds on noise.

## 7. #2 ‚Äî Worked example
`examples/todo-cli/` (documentation, not part of the installed plugin): a filled `docs/plan.md`,
two story files, a sample `tmp/log.md` and `tmp/pm-state.json`, a generated `CLAUDE.md`, and a short
`examples/README.md` noting it is illustrative (not executed). Doubles as a format reference.

## 8. #6 ‚Äî CI + OSS hygiene
- `scripts/validate.sh` ‚Äî portable checks (jq-valid manifests + `hooks.json`; required files present;
  agent/skill frontmatter has `name`+`description`; no third-party-plugin names under
  `plugins/pm-skill/`). Runnable locally and in CI.
- `.github/workflows/validate.yml` ‚Äî runs it on push/PR.
- `CONTRIBUTING.md`, `.github/ISSUE_TEMPLATE/*`, `.github/pull_request_template.md`.
- GitHub Releases for `v0.1.0`, `v0.2.0`, `v0.3.0`.

## 9. Quality & release

## 10. Decisions log
- The enforcement hook is **fail-open and project-scoped** ‚Äî inert for anyone not mid-flight in a PM
  project; kill switch `PM_SKILL_NO_ENFORCE=1`.
- `signed_off` in `tmp/pm-state.json` is the single source the hook and the skill share.
- The worked example lives outside the installed plugin (`examples/`), keeping the plugin lean.
